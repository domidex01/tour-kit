## Strengths

- Highest star count in the category
- Smallest bundle
- Clean, modern API
- Beautiful default UI
- MIT license
- Creator's brand (roadmap.sh) drives attention

## Weaknesses

- Not React-specific — vanilla JS, manual integration
- No official React hooks or wrapper
- Limited to highlighting + simple tours
- Single maintainer with competing priorities (roadmap.sh is the main project)
- Maintenance slowed in the last year

## TourKit wins

- Native React hooks + components
- Headless AND composable
- 8 packages vs single-purpose tours
- TypeScript strict
- WCAG 2.1 AA
- shadcn/ui native

## How to position against it

- **Avoid attacking** Driver.js — it's well-respected and MIT. Position as complementary.
- Pattern: "If you want vanilla JS, Driver.js is a good choice. If you want React-native hooks + a full onboarding toolkit, TourKit."
- Code sample: integrating Driver.js in a React app requires refs and effects; TourKit is just hooks.
