## Stats

| License | MIT |

| Framework | **Next.js ONLY** |

## Strengths

- Purpose-built for Next.js App Router
- Designed for shadcn/ui + Tailwind
- Framer Motion animations (nice visuals out of the box)

## Weaknesses

- **Next.js only** — no Vite, CRA, Remix support
- Tiny community
- Tours / tooltips only (no extended features)
- Tight coupling to Framer Motion (hard to swap animation library)
- Not headless
- No TypeScript strict mode

## TourKit wins

- Works with any React setup (Next.js, Vite, CRA, Remix, custom)
- Headless
- 7 Pro packages (checklists, analytics, announcements, adoption, media, scheduling, surveys)
- No Framer Motion dependency
- shadcn/ui compatible but not locked to shadcn

## How to position against it

- **This is the most dangerous new competitor** — closest stack alignment with TourKit's ICP
- Differentiate on **breadth** (not just tours) and **framework flexibility** (not just Next.js)
- Side-by-side demo: same tour, Vite vs Next.js. TourKit runs both; Onborda only the Next.js one.
- Migration path matters — eventually Onborda users will want extended features TourKit has
