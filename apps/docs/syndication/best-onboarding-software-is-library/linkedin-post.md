The onboarding tools market is $3.5 billion. Most of that money flows to SaaS platforms charging $249-$299/month with MAU-based pricing.

Here's what changed: React libraries now ship the same core functionality for $0 and under 12KB gzipped, while SaaS scripts inject 50-200KB of JavaScript on every page load.

The standard "build vs buy" analysis assumes building from scratch. But using an open-source library eliminates ~80% of the custom build effort. It's a third option that rarely gets discussed.

Three trends are accelerating the shift toward code-owned onboarding:

1. The EU Data Act (effective September 2025) adds data portability requirements targeting vendor lock-in
2. AI personalization works better when you own the code and can integrate your own models
3. The headless UI movement (Radix, shadcn/ui) proved that separating logic from UI gives teams more control
