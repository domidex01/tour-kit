React Joyride — the most popular product tour library for React (673K weekly downloads) — was declared "discontinued" by automated health checkers and competitor blogs.

Then V3 shipped as a complete rewrite.

The backstory: the solo maintainer went silent for 4 months while rebuilding the library from scratch. No communication, no beta channel. During that gap, React 19 broke V2, Snyk flagged it, and competitors published "alternatives" articles.

Three releases in nine days later, V3 landed with React 19 support, Floating UI integration, a modern hooks API, and a 30% smaller bundle.

This pattern — automated health scoring creating a "declared dead" feedback loop for libraries that are actually in active development — is worth paying attention to. It affects how engineering teams evaluate open-source dependencies.
