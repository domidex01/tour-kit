## Thread (6 tweets)

**1/** Self-service costs $0.10 per contact. Live chat costs $8-12.

That's an 80x difference, and it's why every B2B SaaS app needs a resource center.

But most teams confuse "resource center" with "knowledge base." They're not the same thing.

**2/** A knowledge base = external doc site (deep reference, good for SEO)

A help center = KB + FAQs + contact forms + forums

A resource center = in-app widget that pulls contextual help based on what the user is doing RIGHT NOW

The difference is where it lives and how it targets content.

**3/** Here's the problem with knowledge bases alone:

43% of self-service cases fail because users can't find the right content (Gartner).

Resource centers fix this by showing billing docs on the billing page, not your entire 200-article library.

**4/** The five layers of a resource center:

- Trigger (floating button with aria-expanded)
- Panel (slide-over with focus trap)
- Search (client-side fuzzy matching)
- Sections (collapsible content groups)
- Targeting (segment rules by user/page/behavior)

**5/** Build vs buy:

Buy: ships fast, costs $200-800/mo per seat, injects third-party scripts
Build: full design control, predictable costs, requires React devs

Headless libraries sit in the middle — targeting logic without per-seat fees.
