## Title: Open source onboarding: what $0 actually costs in developer time

## URL: https://usertourkit.com/blog/open-source-onboarding-cost-developer-time

## Comment to post immediately after:

I built an open source onboarding library (Tour Kit) and wrote up the actual cost analysis of "free" product tour libraries vs. SaaS tools vs. code-owned solutions.

Key findings: At typical US dev rates, the first year of using a free tour library runs $10,000-$20,000 when you factor integration (40-80 hours), monthly maintenance (5-10 hours for selector updates and framework patches), and the custom builds you'll need for analytics and checklists.

Some broader data: the 2024 Tidelift report found 60% of OSS maintainers work unpaid and 44% cite burnout. Specific to tour libraries, Shepherd.js has an open React 19 compat issue, React-Shepherd's repo is no longer maintained separately, and react-virtualized hasn't released since December 2020.

I tried to be honest about the counterargument too — React Joyride is genuinely good for MVPs and simple use cases. The cost argument applies when requirements grow past basic tooltips.

The article includes a full TCO comparison table. My obvious bias: I built one of the options being compared, and I note that in the article.
