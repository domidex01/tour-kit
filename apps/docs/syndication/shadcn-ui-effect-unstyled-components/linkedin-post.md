shadcn/ui reached 104,000 GitHub stars in under 3 years. That's faster than most frameworks, let alone component libraries.

But the interesting part isn't the adoption curve. It's what shadcn/ui proved about how engineering teams actually want to work with UI code.

The unstyled component model won — and it's reshaping how we think about every UI layer, from buttons to product tours.

Three forces drove the shift:
→ Design systems diverged (every team maintains its own tokens)
→ React Server Components broke runtime CSS-in-JS
→ AI coding tools need source code they can actually read and modify

Radix UI now pulls 9.1M weekly npm downloads. MUI shipped Base UI 1.0 with 35 accessible components. Adobe's React Aria covers 50+ components with 30+ language i18n. The headless ecosystem crossed 14M combined weekly installs.

The takeaway for engineering leaders: the "behavior as dependency, presentation as owned code" pattern applies beyond generic UI. It applies to onboarding, tours, notifications — any interactive layer where your design system matters.
