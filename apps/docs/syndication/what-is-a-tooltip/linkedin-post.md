Did you know that `role="tooltip"` has no meaningful effect on screen readers?

I spent a week writing a proper developer definition of tooltips and found some things that surprised me:

The W3C's own tooltip pattern spec still lacks task force consensus in 2026. WCAG 1.4.13 requires three properties (dismissable, hoverable, persistent) that most custom implementations miss. And 59% of web traffic is mobile — where hover-only tooltips are invisible.

Most "what is a tooltip" articles are written by product marketers. I wrote one for engineers — covering the four functional types, ARIA implementation patterns, and how tooltips work differently inside product tours vs standard UI.
