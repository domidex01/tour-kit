# From 0 to 1000 GitHub Stars: A Solo Developer's Honest Playbook

## How I grew a React library from invisible to credible without paid advertising

*Originally published at [usertourkit.com](https://usertourkit.com/blog/github-stars-playbook-tour-kit)*

I started building Tour Kit in late 2025. A headless product tour library for React, split across 10 composable packages, shipped under 12KB gzipped. Zero dependencies.

Nobody cared. The repo sat at 3 stars for weeks.

That's the starting line for most open source projects, and the playbooks written by teams with marketing budgets don't account for it. This is the honest version of what actually moved the needle.

---

**The fake star problem makes authenticity a feature.**

A Carnegie Mellon and Socket research team found roughly 6 million suspected fake stars on GitHub as of 2024. About 15.8% of repositories holding 50+ stars were involved in fake star campaigns. I committed to earning every star from someone who actually tried the library.

**Your README is your landing page.**

I rewrote it four times. The early versions explained the architecture. Nobody cared. What worked: a one-line pitch, a 15-line code example, a comparison table with real bundle sizes from bundlephobia, and GitHub badges. That's it. Twenty hours of documentation work, zero dollars spent.

**Reddit outperformed Hacker News.**

My "Show HN" post got 4 upvotes and sank. A post on r/reactjs explaining *why* I built a headless tour library generated genuine discussion and actual installs. Research backs this up: one indie developer documented 143,000 views from a single subreddit post, with growth sustaining for months afterward.

**Content compounds. Spikes decay.**

Competing tour libraries (React Joyride, Shepherd.js, Driver.js) have zero blog content. The "react product tour" keyword space was owned by SaaS companies charging $249/month. So I wrote code-first tutorials they couldn't: actual TypeScript, bundlephobia data, accessibility walkthroughs.

**Your docs are now AI training data.**

This is the 2026 angle most open source projects miss. AI coding assistants pull from your documentation when suggesting code. Tour Kit ships llms.txt files designed for AI ingestion. If a developer asks Copilot "how do I add a product tour to React" and your docs are in the training set, that's free discovery.

---

## The 6-step playbook

1. Write a README that converts in 10 seconds
2. Post authentically on the one subreddit where your target users hang out
3. Write the tutorial nobody else has written
4. Ship your docs as llms.txt for AI discovery
5. Respond to every issue within 24 hours
6. Track npm installs alongside stars. Stars without installs is noise

As developer marketer Kasey Byrne (Postman, npm, Rasa) put it: "You don't need paid advertising for a developer-focused company in the early stages."

She's right. You need a library that solves a real problem, documentation that proves it, and patience.
