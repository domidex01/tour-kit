There are 6 million suspected fake stars on GitHub.

That's the finding from a Carnegie Mellon and Socket research team. About 15.8% of repositories with 50+ stars were involved in fake star campaigns. 60% of those repos turned out to be phishing or malware.

I built an open source React library (Tour Kit) as a solo developer. Here's what I learned growing it from 0 to 1,000 authentic GitHub stars with zero paid advertising:

Your README is your landing page. I rewrote it four times. Nobody cared about the architecture. What converted: a one-line pitch, a 15-line code example, and a comparison table with real numbers from bundlephobia.

Reddit outperformed Hacker News. My "Show HN" got 4 upvotes. A post on r/reactjs explaining *why* the library exists generated real discussion and actual installs.

Content compounds. Every competing tour library has zero blog content. I wrote the tutorials SaaS companies charging $249/month couldn't write: actual TypeScript, real benchmarks, accessibility walkthroughs. Those articles still drive traffic months later.

Your docs are now AI training data. In 2026, shipping llms.txt files so AI coding assistants can discover your library is a genuine new growth channel.
