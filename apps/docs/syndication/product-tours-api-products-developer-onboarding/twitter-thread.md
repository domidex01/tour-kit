## Thread (6 tweets)

**1/** I analyzed how Stripe, Twilio, and Postman onboard developers.

The #1 metric they all track: TTFC (Time to First Call).

Champion-level: under 2 minutes → 3-4x higher adoption.
Red flag: over 10 minutes → 50-70% quit rate.

Here's what I found 🧵

**2/** 82% of organizations are now API-first.

But here's the disconnect: most API companies invest heavily in docs and sandboxes while completely ignoring the dashboard experience.

Developers spend real time generating keys, configuring webhooks, and switching environments. That's where friction lives.

**3/** Stripe's "7 lines of code" philosophy is a product decision, not a marketing line.

Interactive code samples in 7+ languages. Run in the browser. No local setup.

Result: docs became their primary sales channel.

**4/** Postman tested providing ready-to-run collections alongside traditional docs.

TTFC dropped 1.7x from a 17-minute baseline.

Some APIs saw 56x improvement.

Pre-built working requests > blank canvas for developers.

**5/** The biggest gap I found: nobody talks about the sandbox → production transition.

That's where most developer churn happens. Key rotation, webhook verification, rate limits, error handling.

A contextual product tour at this moment has the highest ROI of any onboarding investment.
