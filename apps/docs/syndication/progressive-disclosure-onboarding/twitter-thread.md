## Thread (6 tweets)

**1/** Jakob Nielsen introduced progressive disclosure in 1995. Thirty years later, most SaaS apps still dump 10 features on users during their first login.

Data from 15M interactions: linear tours hit 53% completion. Progressive disclosure hits 75%.

Here's the three-layer framework that actually works:

**2/** Layer 1: Orientation (first session)

One guided action. Not five. Not ten. One.

A welcome modal collects role/use case, then one tooltip points at the primary action. The restraint is the entire point.

**3/** Layer 2: Contextual guidance (first week)

Hotspots appear when users navigate to new screens, gated by behavior.

Created 3+ items? Now you see bulk actions.
Invited a teammate? Now you see collaboration features.

Calendar-based drip campaigns aren't progressive disclosure. They're scheduled interruptions.

**4/** Layer 3: Power user features (weeks 2+)

Checklists for keyboard shortcuts, API access, and automation. Only shown to users who've hit usage thresholds.

Nobody sees the automation checklist on day one. By the time it appears, the user has context to want it.

**5/** The metrics most teams miss: track EACH layer independently.

- Trigger rate (who sees it?)
- Completion rate (who finishes it?)
- Activation correlation (does it drive the outcome?)
- Time to value (how long until independent use?)
