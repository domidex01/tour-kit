# Progressive disclosure in onboarding: the right amount of information

## Why showing users everything at once kills activation (and the three-layer fix)

*Originally published at [usertourkit.com](https://usertourkit.com/blog/progressive-disclosure-onboarding)*

Jakob Nielsen introduced progressive disclosure in 1995 to solve a tension that hasn't gone away: users want powerful software, but they don't want to learn it all at once. Thirty years later, SaaS onboarding still gets this wrong.

Most product tours dump every feature into a linear walkthrough. 47% of users skip the tour entirely.

Progressive disclosure flips that approach. Show the core action first. Surface secondary features when the user reaches them. Reveal advanced capabilities once someone has demonstrated they need them.

The data backs this up. Companies using contextual onboarding with progressive disclosure boosted completion rates to 75% and saw a 30% increase in paid conversions. Compare that to the 53% average completion for linear product tours across 15 million interactions.

## The three layers

**Layer 1: Orientation (first session)** — Answer one question: "What's the single most important thing I should do right now?" One guided action. Not five. One.

**Layer 2: Contextual guidance (first week)** — Hints and hotspots appear next to features the user hasn't discovered yet, but only when they navigate to the relevant screen. Triggered by behavior, not a calendar.

**Layer 3: Power user features (weeks 2+)** — Keyboard shortcuts, API access, automation rules. These appear through checklists or prompts that reference what the user has already accomplished. Nobody sees the automation checklist on day one.

## Where teams go wrong

The most common mistake is treating progressive disclosure as "hide everything." It means making the most important action obvious and everything else discoverable. Nielsen's research specifically warns that more than 2 disclosure levels causes navigation confusion.

The second mistake is time-based drip campaigns. Showing feature X on day 3 regardless of what the user has done isn't progressive disclosure. It's scheduled interruptions. Real progressive disclosure is behavior-triggered.

## Measuring it

For each disclosure layer, track four things: trigger rate (what percentage of eligible users see this layer), completion rate, activation correlation, and time to value.

Targets: Layer 1 should hit 75%+ completion. Layer 2 should reach 50%+. Layer 3 is about retention, not activation, with 40%+ completion from the 20-30% of users who qualify.

If your Layer 1 completion is below 60%, the orientation step is asking too much. Cut it down.
