Your open-source project probably has a contributor problem.

Not a "nobody knows about it" problem — a "people use it but never contribute" problem. Homebrew's maintainer describes the typical funnel: millions of users, thousands of contributors, tens of maintainers.

GitHub added 36 million new developers in 2025. But the gap between using open-source code and contributing to it keeps widening.

The data shows what helps: README files (+55% productivity), contribution guidelines (+17%), mentorship (+46%). But less than 1% of projects use interactive walkthroughs for contributor onboarding.

I wrote a guide on using product tours to close that gap — walking new contributors through the codebase, dev setup, and PR process before they submit their first PR. Especially relevant with the "AI slop" problem (AI-generated low-quality contributions flooding maintainers).
