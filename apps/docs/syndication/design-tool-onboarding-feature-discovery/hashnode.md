# Design tool onboarding: feature discovery in complex UIs

Design tools pack hundreds of features into a single canvas. Figma has vector editing, auto layout, dev mode, prototyping, variables, and components, all accessible from one screen. The onboarding problem isn't "show the user where the button is." It's "reveal the right capabilities at the right moment without overwhelming a user who just wants to draw a rectangle."

This guide breaks down how Figma, Canva, Adobe XD, Sketch, and Penpot handle feature discovery, what progressive disclosure patterns work in complex UIs, and how to implement similar onboarding in React using Tour Kit.

Key data points covered:
- Optimal tour length: 3-5 cards (Slack uses 4, Canva uses 1-3, Notion uses 6)
- Developers with intuitive tools feel 50% more capable of creative problem-solving
- WCAG 2.1.4 requirements for keyboard shortcut onboarding
- Side-by-side comparison of 5 design platform onboarding approaches
