## Thread (6 tweets)

**1/** Developers with intuitive tools feel 50% more capable of creative problem-solving (GitHub/Microsoft/DX study).

So how do design tools with 200+ features actually teach you to use them?

I compared Figma, Canva, Adobe XD, Sketch, and Penpot.

**2/** Figma: Opt-in 4-5 card tour with animated tooltips. Respects your choice to skip.

Weakness: hasn't evolved since 2022. A Sketch veteran sees the same beginner tips as someone opening a design tool for the first time.

**3/** Canva: 1-3 contextual cards triggered by YOUR actions, not by login.

Click text for the first time? Card explains font pairing. Resize an image? Hint surfaces the background remover.

Tradeoff: power features stay invisible to users who never trigger the right context.

**4/** The pattern that works: progressive disclosure with 3-5 cards max per micro-tour.

Slack: 4 cards. Canva: 1-3. Notion: 6. Nobody goes above single digits anymore.

Key insight: trigger based on behavior, not time.

**5/** The gap nobody's filling: no platform combines all three:
- Opt-in (not forced) short tours
- Behavior-triggered progressive disclosure
- Full WCAG 2.1 AA accessibility

That's where headless tour libraries come in.
