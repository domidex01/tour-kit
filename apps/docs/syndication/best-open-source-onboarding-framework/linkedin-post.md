Every "best onboarding tools" list I could find was written by a SaaS vendor. So I tested every open-source option myself.

The biggest takeaway: most developers search for an "onboarding framework" but get pointed toward tooltip libraries. These are fundamentally different things.

Tour libraries render tooltips on DOM elements. Onboarding frameworks coordinate the full lifecycle — tours, checklists, surveys, analytics, fatigue prevention. The gap between the two is where engineering teams spend $6,000+/year in integration time.

Some surprising findings from the comparison:
- React Joyride (400K+ weekly npm downloads) has no stable React 19 release
- Intro.js uses AGPL v3, which many teams discover after integration
- No single open-source library covers the full onboarding stack

I wrote up the full comparison with bundle sizes, licensing analysis, and a decision framework: https://usertourkit.com/blog/best-open-source-onboarding-framework

(Disclosure: I built Tour Kit, one of the frameworks in the comparison. Limitations included.)
