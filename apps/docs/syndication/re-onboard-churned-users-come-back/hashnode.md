# How to re-onboard churned users who come back

Your marketing team spent weeks crafting the win-back email sequence. The user clicked. They logged back in.

Now what?

Most products greet returning users with the same onboarding flow they saw months ago. That's the fastest way to lose them a second time. Re-onboarding is a different problem than onboarding, and this guide covers how to detect returning churned users, segment them by churn reason, and build conditional tour flows in React.

Full article with all code examples and comparison table: [usertourkit.com/blog/re-onboard-churned-users-come-back](https://usertourkit.com/blog/re-onboard-churned-users-come-back)

Key takeaways:

- **43% of churn** stems from unclear onboarding steps (UserGuiding, 2026) — many "churned users" never actually onboarded
- The optimal win-back window is **2–6 weeks** post-churn (Hightouch)
- Event-based triggers outperform calendar-based campaigns **3–5x**
- Adaptive onboarding that skips known steps improves completion by **35%**
- Track **60-day retention** of reactivated users, not tour view count

The article includes working TypeScript code for: returning user detection (localStorage + server-side), churn-segment routing into different tour variants, changelog-driven tours filtered by `lastActiveDate`, and adaptive step filtering.
