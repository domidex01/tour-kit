**1/** 62% of new signups never returned after day one on our analytics dashboard.

Then we replaced the 12-step tooltip tour with 3-step role-based tours.

7-day retention jumped 34%.

Here's what we learned about complex dashboard onboarding:

**2/** Three-step tours hit 72% completion. Past five steps, it collapses.

Working memory holds ~7 items for 30 seconds. Only 20% of users read page content.

Dashboard onboarding has to work within those limits, not fight them.

**3/** Pattern 1: Role-based tour routing.

A finance analyst and an ops manager have completely different aha-moments in the same dashboard.

One signup question. Three steps per role. Each step connects to an action, not just a label.

**4/** Pattern 2: Empty states ARE your onboarding.

Before any data exists, you have a full screen with zero cognitive noise. Warm copy + single CTA + skeleton preview of what populated looks like.

Stripe nails this.

**5/** Pattern 3: Everboarding.

Don't tour everything on day one. Fire contextual micro-tours the first time a user visits each section.

Linear does this with their command palette — it appears only when readiness signals show up.
