**2/** The problem: Tour Kit is a React library. It ships in your bundle, runs locally, doesn't phone home.

There's no server. No dashboard. No per-user infrastructure.

Charging monthly for code sitting in node_modules felt like charging rent on furniture you already own.

**3/** The data confirmed the instinct:

- 78% of finance decision-makers are scrutinizing recurring costs more
- 62% of businesses are consolidating SaaS subscriptions
- The median dev tool costs $32/user/month ($384/year)

$99 one-time = ~3 months of a typical tool subscription.

**4/** The hardest part wasn't the price. It was the free/paid boundary.

Core packages (tour logic + React components + hints): MIT, free forever, functionally complete.

Pro packages (surveys, scheduling, analytics, checklists): $99 one-time.

Adjacent capabilities, not gated core features.

**5/** What I got wrong:

- Should have started at $149 and run $99 launch sales (pricing down > pricing up)
- Should have set activation limits from day one
- Should have written about this decision months earlier

**6/** The honest tradeoff: no MRR means starting from zero every month.

But "pay once, use forever" stops people mid-scroll in a world where every tool wants a monthly relationship with your credit card.

Subscription #131 doesn't get noticed. Permanence does.
