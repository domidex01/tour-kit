The open-source services market hit $49 billion in 2025. But most developer library maintainers still work for free.

I built a React library as a solo developer and had to choose between three business models: open core, dual licensing, and open-source SaaS. Each one makes different tradeoffs between adoption speed and revenue.

The framework that helped most: "buyer-based segmentation" from Open Core Ventures. Don't ask "how hard was this to build?" Ask "who benefits most from this feature?" Features for individual contributors stay free. Features for product managers and VPs go paid.

Two data points that shaped my thinking:
- 96% of organizations maintained or increased open source usage in 2025 (Linux Foundation)
- Postiz, a solo developer's open-source project, generates $14.2k/month

The tooling has finally caught up too. Polar.sh handles license keys and payments for 4% + 40c per transaction. No monthly fees. Open source itself.

Full breakdown with pricing tiers and five rules for monetizing a developer library: https://usertourkit.com/blog/open-source-business-model-developer-libraries
