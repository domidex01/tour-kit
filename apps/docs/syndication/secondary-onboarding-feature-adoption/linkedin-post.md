75% of SaaS features go undiscovered.

Not because they're bad features. Because users never find them after the initial onboarding flow ends.

The average SaaS feature adoption rate is 24.5% (median 16.5%). Users who adopt features regularly are 31% less likely to churn. Yet 36% of companies still have zero intentional in-app guidance.

I wrote a guide on secondary onboarding — the post-activation phase where you surface advanced features to existing users. It covers:

- Behavioral triggers (gate on user actions, not calendar time)
- Progressive feature disclosure as a React component pattern
- User segmentation without needing a CDP
- The TARS measurement framework for deciding which features need help

Every existing article on this topic treats it as a no-code PM problem. This is the code-first angle for engineering teams who want onboarding logic version-controlled and reviewed alongside feature code.
