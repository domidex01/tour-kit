## Thread (6 tweets)

**1/** Hot take: Onboarding is a developer problem, not a product manager problem.

Every no-code onboarding vendor says "ship tours without developers." Then when the tour breaks production, guess who gets paged? 🧵

**2/** The accountability gap is real.

PMs configure tours in visual builders. Those tours inject 50-500KB of third-party JS at runtime.

The PM never sees the Lighthouse score drop. The developer does.

**3/** WCAG 2.1 SC 4.1.2 says the quiet part out loud:

"This success criterion is primarily for web authors who develop or script their own user interface components."

Product tours ARE custom UI components. Accessibility compliance is structurally a developer's job.

**4/** The "build vs. buy" argument is outdated.

It assumes "build" costs $50-150K in custom engineering. Open-source headless libraries collapse that assumption.

The real choice: SaaS subscription vs. code ownership at near-zero cost.

**5/** To be fair, no-code tools have legitimate advantages.

Teams iterating weekly see 25% higher activation. That's real.

But the bottleneck isn't code — it's sprint cadence. CI/CD + feature flags solve iteration speed without ceding ownership.
