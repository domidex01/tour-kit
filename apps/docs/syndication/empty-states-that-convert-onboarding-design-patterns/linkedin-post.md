Your new users are hitting a wall you probably haven't designed for.

The blank dashboard. No data, no projects, no guidance. According to SaasFactor research, 75% of users abandon within the first week when onboarding fails at this moment.

I wrote a guide on empty state design patterns that convert, covering four approaches that move activation numbers:

1. The guided action pattern (what Shopify Polaris and IBM Carbon standardize on)
2. Demo data with context labels (how Autopilot cut their 50% trial churn)
3. Milestone trackers embedded in the empty state itself
4. Conversational CTAs that tailor the downstream experience

The biggest gap I found: every UX article on empty states cites the same 2017 Smashing Magazine piece, and none cover accessibility. WCAG 4.1.3 requires empty state transitions to be announced to screen readers, but zero of the top results implement aria-live.
