`SurveyProgress` renders a question counter and/or progress bar for multi-step surveys. It returns `null` when `total` is 1 or less â€” no progress indicator is needed for single-question surveys.

## Import

```tsx
import { SurveyProgress } from '@tour-kit/surveys';
```

## Usage

```tsx
import { SurveyProgress, useSurvey } from '@tour-kit/surveys';

function SurveyWithProgress() {
  const survey = useSurvey('multi-step-survey');
  const currentStep = (survey.state?.currentStep ?? 0) + 1;
  const totalSteps = survey.config?.questions.length ?? 1;

  return (
    <div>
      <SurveyProgress
        current={currentStep}
        total={totalSteps}
        mode="both"
      />
      {/* question content */}
    </div>
  );
}
```

### Text only

```tsx
<SurveyProgress current={2} total={5} mode="text" />
// Renders: "Question 2 of 5"
```

### Bar only

```tsx
<SurveyProgress current={3} total={5} mode="bar" size="sm" />
```

### Custom label template

```tsx
<SurveyProgress
  current={2}
  total={4}
  labelTemplate="Step {current} / {total}"
/>
// Renders: "Step 2 / 4"
```

## Variants

`SurveyProgress` uses `progressBarVariants` from CVA for the filled bar:

| Variant | Values | Default |
|---------|--------|---------|
| `size` | `'sm'` `'md'` `'lg'` | `'md'` |

Bar heights: `sm` â†’ `h-1`, `md` â†’ `h-2`, `lg` â†’ `h-3`.

## Accessibility

- Progress bar: `role="progressbar"`, `aria-valuenow={current}`, `aria-valuemin={1}`, `aria-valuemax={total}`, `aria-label="Survey progress"`
- Text label: rendered in a `<span aria-live="polite">` so screen readers announce step changes
